# 돌림판 제작 (Roulette Maker)

아래 **"⚙️ 설정"** 칸의 값을 읽어 참고 이미지(`assets/reference.png`)의 구조를 참고한 인터랙티브 돌림판 HTML 을 만든다.

> 사용법: 아래 **"⚙️ 설정"** 칸의 `{ }` 안 값을 원하는 대로 고친 뒤 `"돌림판 만들어줘"` 라고 하면 된다.

---

## ⚙️ 설정 (여기를 직접 수정하세요)

### 1단계 — 칸 수 (버튼 2개)
- 버튼1 칸수 : { 6 }
- 버튼2 칸수 : { 35 }

### 2단계 — 색상
- 색상 : { 초록색 }

### 3단계 — 제목
- 제목 : { 3반 돌림판 }

---

## 설정 값 해석 규칙

`{ }` 안의 값을 아래 규칙으로 해석한다. 값이 비었거나 `{ 수정해 보세요 }` 같은 안내문 그대로면 기본값을 쓴다.

- **버튼1 칸수 / 버튼2 칸수** — 각각 2 이상의 정수. 기본값 `6`, `32`. 돌림판 상단(제목 배너 아래)에 이 두 값으로 만든 **버튼 2개**를 놓고, 참고 이미지처럼 `1 ~ N` 형태로 표기한다(예: `1 ~ 6`, `1 ~ 32`). 버튼을 누르면 그 칸 수로 돌림판을 다시 그린다. 처음에는 버튼1이 선택된 상태로 보여 준다. 두 값이 같거나 한쪽만 채워졌으면 채워진 값으로 하나의 버튼만 만든다. 40 초과 값은 글자가 겹칠 수 있으니 그대로 만들되 한 줄 안내한다.
- **색상** — 아래 테마 이름 중 하나이거나, 색상 코드(`#RRGGBB`)를 공백/쉼표로 나열한 목록. 기본값 `무지개`.
  - **무지개** — `#e74c3c #3498db #2ecc71 #f39c12 #9b59b6 #e67e22 #1abc9c #e84393`
  - **파랑** — `#1e5799 #2989d8 #3498db #5dade2 #85c1e9 #2874a6`
  - **빨강** — `#c0392b #e74c3c #ff6b6b #e84393 #fd79a8 #d63031`
  - **초록** — `#145a32 #1e8449 #27ae60 #2ecc71 #58d68d #82e0aa`
  - **파스텔** — `#ffadad #ffd6a5 #fdffb6 #caffbf #9bf6ff #a0c4ff #bdb2ff #ffc6ff`
  - **골드** — `#2c2c2c #c9a227 #3d3d3d #e0b83d #1a1a1a #b8860b`
  - 위 이름이 아니면 적힌 색만으로 배열을 만든다. 인접 칸이 같은 색이 되지 않도록 색 개수를 칸 수와 서로소가 되게 하거나 충분히 많게 보정한다.
- **제목** — 상단 배너와 브라우저 탭 제목(`<title>`)에 쓴다. 기본값 `행운의 돌림판`.

## 제작 방법

1. 위 **"⚙️ 설정"** 칸의 값을 읽고 위 규칙으로 해석한다. (문답 없이 바로 진행)
2. `assets/reference.png` 를 Read 로 열어 디자인(레이아웃·구성)을 확인한다.
3. 해석한 값(칸 수·색상·제목·항목)을 반영하여 **단일 HTML 파일을 직접 작성**한다.
   외부 라이브러리·CDN 없이 순수 HTML/CSS/JS 로만 만들어 더블클릭으로 바로 열리게 한다.
4. 완성한 파일을 사용자의 작업 폴더에 `roulette.html`(또는 제목 기반 파일명)로 `Write` 한다.
5. 어떤 설정값으로 만들었는지 요약하고, 저장 경로와 사용법(중앙 "돌리기" 또는 "돌리기" 버튼 클릭)을 알려준다.
6. **(주의) 검증 제한**: 본 스킬은 정적 HTML 단일 파일 제작을 목적으로 합니다. 검증(Verification) 단계를 진행할 때 로컬 웹 서버를 기동하거나 자동화 브라우저 도구(browser_subagent)를 구동하여 `file:///` 또는 `localhost` 접속 테스트를 시도하지 마십시오. 오직 작성된 코드의 구문과 로직만 정적으로 자체 검증한 뒤 완료합니다.

## 디자인·기능 요구사항 (참고 이미지 기준)

참고 이미지(`assets/reference.png`)의 구조를 따르되, 아래 요소를 반드시 포함한다.

- **배경**: 색 배열의 가장 밝은 색보다 한톤 더 밝은색으로 지정 + 해당색과 어울리는 테마의 배경 Confetti. 
  - 예시 : 핑크색 + 벛꽃 Confetti / 파란색 + 구름 Confetti
- **상단 제목 배너**: 골드색 화살표(리본) 모양 배너에 제목. 배너 위아래로 알록달록 장식 조명(깜빡임).
- **칸수 전환 버튼 2개**: 배너 아래에 `1 ~ N` 형태 버튼 2개(예: `1 ~ 6`, `1 ~ 32`). 클릭하면 해당 칸 수로 돌림판을 즉시 다시 그리고, 선택된 버튼은 강조(골드 배경) 표시한다. 회전 중에는 전환을 막는다. 참고 이미지의 버튼 위치·모양을 따른다.
- **돌림판(canvas 권장)**: N 칸의 부채꼴을 색 배열로 순환 채색, 각 칸 중앙에 라벨(흰색, 그림자). 골드 이중 테두리.
- **포인터**: 돌림판 상단(12시)에 아래를 향하는 빨간 삼각형 화살표(고정).
- **중앙 허브**: 흰 원 안에 "돌리기" — 클릭하면 회전.
- **우측 결과 패널**: "★ 결과 ★" 라벨과 당첨 라벨을 강조 표시하는 상자.
- **회전 버튼**: 골드색 "돌리기" 버튼.

### 회전 로직 (정확성 필수)
- 클릭 시 무작위(`Math.random`)로 당첨 칸을 먼저 정한 뒤, **그 칸의 중앙이 상단 포인터에 정확히 멈추도록**
  목표 회전 각도를 역산한다. (여러 바퀴 + 감속 이징으로 자연스럽게)
- **(치명적 주의) 정수형 회전수 강제**: '여러 바퀴'에 해당하는 회전 횟수 `K`는 반드시 정수(Integer)형이어야 최종 정지 시 휠의 정렬이 어긋나지 않습니다. 실수형(Float) 난수를 그대로 사용할 경우, 최종 각도가 틀어져 포인터 지점과 결과 텍스트가 일치하지 않는 치명적인 버그가 발생합니다. 반드시 `Math.floor` 등을 사용해 정수 계수로 만드십시오.
- 정지 시점에 포인터가 가리키는 칸과 결과 패널에 표시되는 값이 **반드시 일치**해야 합니다. 애니메이션 이징이 완전히 끝난 시점에는 휠의 `transform: rotate` 각도가 `targetAngle` 값과 정확히 일치하는지 최종 보정하여 부동소수점 오차를 방지하십시오.
- 각도 계산 요령: 첫 칸(index 0) 중앙을 12시 방향에 배치하면, 당첨 칸 `w`에 멈추기 위한 회전각은
  `K*360 − w*(360/N)` 도 (단, `K`는 정수). 정지 후 `labels[w]` 를 결과로 표시한다.
- 회전 중에는 버튼을 비활성화해 중복 클릭을 막는다.
